XML Rules


XML has rules that must be followed in order to be a valid XML file. Listed below are 6 examples of rules that must be followed.

XML Rule 1

When writing an XML file, at the top of the page, you must delcare that it is an XML file and its version type. Example shown below:

<?xml version...>

XML Rule 2

All XML files must have a root element. This root element is the holder of all other elements in your file and can be named to match your content. For example, if writing a contact list the root element would be:

<contact-list>

XML Rule 3

Unlike HTML, all elements within your XML file must have closing tags. Example shown below:

</contact-list>

XML Rule 4

All elements are case sensitive. For example:

</Contact-List> is incorrect

XML Rule 5

Elements can be assigned attributes. These must be of single-value. For example, a contact within the contact list could have a home or work phone number:

<phone-number category="work">

XML Rule 6

When creating XML documents, you have the ability to create comments within your code. Example shown below:

<!-- Comment goes here -->
